# Documentation Contribution Guide

Guidelines for contributing to Workflow Engine documentation.

## Documentation Structure

Documentation is organized in the `docs/` directory:

- `getting-started/` - Getting started guides
- `guides/` - User and developer guides
- `api/` - API reference
- `contributing/` - Contribution guides

## Writing Documentation

### Style Guidelines

- **Clear and concise**: Use simple, direct language
- **Code examples**: Include working code examples
- **Links**: Link to related documentation
- **Formatting**: Use consistent markdown formatting

### Code Examples

- **Test examples**: Ensure all code examples work
- **TypeScript**: Use TypeScript for all examples
- **Comments**: Add comments to explain complex code
- **Complete examples**: Provide complete, runnable examples

## Documentation Types

### Getting Started Guides

- Step-by-step instructions
- Complete examples
- Clear explanations

### API Documentation

- Method signatures
- Parameter descriptions
- Return value descriptions
- Usage examples

### Guides

- Clear explanations
- Practical examples
- Best practices
- Common pitfalls

## Submitting Documentation

1. **Create branch**: Create feature branch
2. **Make changes**: Edit documentation files
3. **Review**: Review for clarity and accuracy
4. **Submit PR**: Submit pull request
